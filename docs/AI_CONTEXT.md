## 项目目标

Crypto Pulse 目标是建立一个 Telegram 加密货币社区生态，围绕数据服务、社区运营和自动化内容形成长期增长。

核心方向：

- 数据服务：行情播报、价格查询、异动提醒、涨跌榜、热门币排行。
- 社区运营：群互动、用户活跃统计、用户留存、社区活动。
- 自动化内容：市场摘要、新闻摘要、排行榜、异常波动提示。

## 项目定位

当前阶段：MVP 阶段的 Telegram 加密货币工具与频道系统。

目标定位：面向 Telegram 用户的加密货币社区生态平台。

产品策略：

- 从“行情查询工具”逐步转向“社区运营产品”。
- 以免费服务获取真实用户和活跃度。
- 用自动化内容降低人工运营成本。
- 避免堆积大量重复的价格查询命令。

## 盈利模式

未来可能盈利方式：

- 广告收入。
- 项目合作推广。
- 联盟邀请链接。

明确不做：

- 不做喊单。
- 不做信号群。
- 不做付费会员。
- 不提供投资建议。

## 当前架构

### Bot1：广播员 Bot

- 主触发来源：Render 内部广播接口。
- GitHub Actions：自动 schedule 已停用，仅保留手动测试备用。
- 生产运行方式：仍为原每 2 小时排期，等待本地升级部署。
- 本地待部署排期：北京时间 00/04/08/12/16/20 点第 17 分，包含普通行情、早间简报、晚间观察和周日周报。
- 主要功能：自动获取加密货币价格并推送到 Telegram 频道。
- 数据源：CoinGecko API。
- 当前状态：运行中。

### Bot2：客服 Bot

- 部署平台：Render。
- 运行方式：FastAPI Webhook。
- 主要功能：响应 Telegram 群组命令。
- 当前命令：`/price`、`/compare`、`/help`。
- 数据源：CoinGecko API。
- 当前状态：运行中。
- 同群多 Bot 命令隔离：Bot2 会识别 `/command@botusername` 的目标；发给其他 Bot 的命令会静默忽略。
- Bot2 username 通过可选环境变量 `TELEGRAM_BOT_USERNAME_2` 识别，未配置时默认使用当前生产用户名 `CryptoService2_bot`。
- Bot2 对 GroupGuard 专属短命令 `/gid`、`/gst`、`/grules`、`/gabout`、`/gdisc`、`/ghelp`、`/grep` 静默忽略，避免同群多 Bot 冲突。

### 频道

- 状态：已创建，未公开运营。
- 用途：接收自动行情播报和后续自动化内容。

### 群组

- 状态：已创建，未公开运营。
- 用途：用户互动、命令查询、社区讨论。

## 技术限制

- Binance API 因地区限制不可用。
- 优先使用 CoinGecko API 或其他无地区限制的数据源。
- 必须优先使用免费部署方案。
- 当前优先平台：GitHub Actions、Render、免费数据库。
- 数据库起步方案优先 SQLite，未来可迁移 PostgreSQL。
- 项目应保持低成本或零成本运营。

## 开发原则

1. 优先用户增长，不是优先开发复杂功能。
2. 优先社区活跃，不是优先增加查询命令。
3. 优先自动化内容，不是人工维护。
4. 优先免费方案，不是增加服务器成本。
5. 代码必须模块化，避免长期维护困难。
6. 所有重大改动必须同步更新文档。

## 优先级原则

优先做：

- 异动提醒。
- 涨跌榜。
- 热门币排行。
- AI 市场摘要。
- 群互动功能。
- 用户活跃统计。

避免做：

- 大量相似价格查询命令。
- 需要付费服务器的新架构。
- 依赖 Binance API 的功能。
- 带有投资建议、喊单、信号性质的功能。

## 未来方向

短期目标：

- 完成 P0 功能。
- 建立基础数据库和用户统计能力。
- 丰富频道自动化内容。
- 让频道和群组具备公开运营前的基础能力。

中期目标：

- 建立用户活跃体系。
- 增加群互动和排行榜功能。
- 增加自动化市场摘要。

长期目标：

- 建立稳定 Telegram 加密货币社区。
- 支持广告、合作推广、联盟链接等商业化方式。
- 保持免费基础服务和自动化运营能力。
