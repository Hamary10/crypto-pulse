# Roadmap - Crypto Pulse

## 用途

本文件记录 Crypto Pulse 的未来规划，所有 AI Agent 开发前必须先查看本文件，避免重复设计和方向偏移。

## P0：MVP 运营基础

目标：让项目具备公开运营前的基础能力。

已完成：

- 建立 SQLite 最小数据库基础。
- 记录用户基础信息。
- 记录命令使用数据。
- 记录币种查询统计。
- 记录价格快照。
- 实现 `/top` 市值排行。
- 实现 `/trending` 热门币榜。
- 实现 `/gainers` 24 小时涨幅榜。
- 实现 `/losers` 24 小时跌幅榜。
- 实现频道每日涨幅榜推送。
- 实现频道每日跌幅榜推送。
- 实现频道每日热门币榜推送。
- 建立统一 CoinGecko API 模块和缓存。

待完成：

- 实现异动提醒基础版本。本次定时策略优化不包含异动提醒开发。
- 建立自动欢迎系统。
- 完善错误处理和日志。
- 保证所有重大改动同步更新文档。

## P1：社区活跃与留存

目标：让群组开始产生真实互动和用户留存。

- 实现用户活跃统计。
- 实现活跃用户排行榜。
- 优化 `/compare` 命令展示。
- 增加群互动话题或每日投票。
- 建立基础运营数据统计。

暂不做：

- 积分系统。
- 等级系统。
- 勋章系统。
- 竞猜系统。
- 邀请系统。

## P2：自动化内容体系

目标：降低人工运营成本，增强频道内容价值。

- 每日涨跌榜自动推送已完成基础版。
- 热门币榜自动推送已完成基础版。
- 未来可评估新币发现或市场热点提示。

暂不做：

- AI 市场摘要。
- 新闻摘要。
- 新闻系统。

## P3：增长、监控与商业化

目标：支持长期运营、增长和未来变现。

- 建立更完整的数据分析后台。
- 增加监控和告警能力。
- 评估 PostgreSQL 迁移。
- 优化部署稳定性。
- 预留广告、项目合作、联盟推广能力。

## 不优先开发

- 大量相似价格查询命令。
- 付费会员系统。
- 信号群功能。
- 喊单相关功能。
- 依赖 Binance API 的功能。
- 会显著增加服务器成本的方案。

## Roadmap 更新规则

每次开发完成后：

- 已完成事项从对应优先级中标记或移除。
- 新发现的需求必须放入 P0-P3 中合适位置。
- 如果优先级发生变化，必须说明原因。
- 同步更新 `PROJECT_STATUS.md` 和 `CHANGELOG.md`。
